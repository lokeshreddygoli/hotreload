Stop restarting your
Go server by hand.

A lightweight CLI that watches your project, rebuilds on save, and restarts the server — under a second, every time.

$ hotreload --root ./myproject \ --build "go build -o ./bin/server ./cmd/server" \ --exec "./bin/server"
View on GitHub → Install

What it handles

⏱

Debounced saves

Editors fire 3–4 events per save. A 300ms quiet window collapses them into one rebuild.

⚡

Cancelled mid-builds

New change while building? Old build dies immediately. Only the latest state gets built.

🌲

Full process tree kill

Kills the entire process group on restart — no orphan workers holding ports open.

🔄

Crash loop protection

3 fast crashes in a row? Backs off 5s before retrying. No spinning CPU.

📁

Dynamic dir watching

New package folder created at runtime? Picked up automatically, no restart needed.

🪟

Windows support

Job Objects + taskkill /F /T fallback. Same CLI on all platforms.

How it works

1

File watcher

Uses fsnotify recursively. Ignores .git/, vendor/, editor swap files. Only .go, go.mod, go.sum trigger rebuilds.

2

Debouncer

Every event resets a 300ms timer. Handles vim's rename-swap, VS Code's multi-event saves, all of it.

3

Build + run cycle

Each cycle runs under a context.Context. New trigger cancels it — kills the in-progress build, starts fresh. Server output streams live to your terminal.

4

Clean shutdown

Ctrl+C cancels the root context, kills the server process group, exits cleanly. No dangling processes.

Install

Clone
go install
Windows
# clone and build
git clone https://github.com/lokeshreddygoli/hotreload
cd hotreload
make build

# optional: add to PATH
export PATH="$PWD/bin:$PATH"
go install github.com/lokeshreddygoli/hotreload@latest
# PowerShell
git clone https://github.com/lokeshreddygoli/hotreload
cd hotreload
go build -o .\bin\hotreload.exe .

.\bin\hotreload.exe `
  --root .\myproject `
  --build "go build -o .\bin\server.exe .\cmd\server" `
  --exec  ".\bin\server.exe"
